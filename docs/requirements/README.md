# 確定要件の保管庫

リリース済み機能の**確定した要件定義**を `<feature>.md` として置く。開発中の作業領域は
`.kiro/specs/<feature>/requirements.md`（SDD の作業領域）で、機能がリリースされたらここへ
**昇格**する（手順は [`../sdd.md`](../sdd.md) 「確定版への昇格」）。

- 「正（確定版）は `docs/`、作業中は `.kiro/`」という役割分担に従う。
- 基本設計の確定版は [`../design/`](../design/) に置く。
- まだ昇格対象がないため、本ディレクトリは README のみ（初期状態）。
